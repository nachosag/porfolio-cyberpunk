// PROJECT_003

HUMAN CLUSTERING

Desktop app that groups people by affinity across 4 interest dimensions using weighted graph theory. Similarity is computed via Manhattan distance, then Kruskal's algorithm builds an MST. High-cost edges are pruned and BFS derives the final connected-component clusters.

TECHNICAL CHALLENGE

Implementing Kruskal's MST at O(E log E) and designing a SOLID architecture where the grouping engine is fully decoupled from its data source — whether JSON files or manual UI input. 100% test coverage on all core logic.

// PROJECT_004

CDM CALCULATOR

Tool for finding the Minimum Dominating Set in large-scale graphs — an NP-Hard optimization problem. Implements both exact Backtracking with recursion-tree pruning and Greedy Heuristics for fast approximations. Includes an interactive graph editor built from scratch in Java AWT and a JUnit stress testing suite.

TECHNICAL CHALLENGE

Pruning the recursion tree aggressively enough to process dense graph instances that would otherwise exhaust memory. Balancing exact solutions (backtracking) vs. fast approximations (greedy) and exposing both through a visual, interactive interface.

# Engineering Manifesto
 
1. Understand the problem BEFORE
touching the keyboard.
 
2. A system is only as good as its
weakest abstraction.
 
3. Complexity is the enemy.
Simplicity is earned.
 
4. Don't just use libraries.
Know what they're doing.
 
5. Production systems have no mercy.
Your architecture better.

◈

CLEAN ARCHITECTURE

Separation of concerns isn't a suggestion. Layers exist for a reason — and violating them always costs you later.

⊞

SYSTEMS THINKING

Every component is part of a larger system. Understanding dependencies and failure modes before writing line one.

⊿

ALGORITHMIC DEPTH

Knowing when NOT to use a library is as important as knowing how to use one. Understand the algorithm first.

◉

DELIBERATE EXPERIMENTATION

Projects like backend-lab exist because patterns need to be tested, broken, and rebuilt before they're production-ready.

⬡

DATA-CENTRIC DESIGN

The data model is the system. Get the schema wrong and everything built on top of it becomes technical debt.
